An overview of couple of projects I have done over last year, including restaurants page, artist portfolio and other side projects.

Our team was presented with a vision of a website for “Gjøl Kro” from it’s owner Peter in his Client brief. They had no website till then, so the task was to create responsive and accessible website which can be accessed from all types of screens and screen sizes, projects all of Peters visions and visually/functionally reflects restaurants image.

Screenshot from finished website https://mmd.ucn.dk/class/MDE-CSD-S21/10407761/gjolkro/index.html

For project development we chose agile method. User research process consisted of user-categorization, interviews, interview analyzing and thematizing. With the results we were able to categorize user needs. Having information architecture, we started prototyping - designing low-fidelity prototypes and eventually high-fidelity prototypes.

Screenshot from finished website https://mmd.ucn.dk/class/MDE-CSD-S21/10407761/gjolkro/ourservices/ourservices.html

After user testing the prototypes, we got to coding and finished the product. My role in this project was mainly working with user-research, designing and overall help to finish the project in given time. In a month me and my truly talented were able to create requirement checked and sophisticated website for newly opened restaurant.

Screenshot from finished website https://mmd.ucn.dk/class/MDE-CSD-S21/10407761/gjolkro/ourstory/ourstory.html

In this assignment we had free choice of project. We decided to go more artistic way choosing to create portfolio. We found a local artist who was in a need for portfolio and a great help for us in this project. We did three user tests including user-research with quantitative and qualitative so that we could see this project from all angles.

Screenshot from finished website https://mmd.ucn.dk/class/MDE-CSD-S21/10407761/connorfernandez/index.html

After gathering loads of data about artist portfolios and our artist we were able to make structure for data, from which we started to create many iterations of wireframes and prototypes. 5-second test was used to prove our design. My main role in this project was to help gather user-research and help with design and sites content.

Screenshot from finished website https://mmd.ucn.dk/class/MDE-CSD-S21/10407761/connorfernandez/contact/get-in-touch.html

This was one of the most fun projects I have taken part of. With a lot of work and enthusiasm we were able to create a website for musician. On the website we have included discography, gallery, contacts, about me and home page - we also created identity & branding which was made from gathered user research. Duration - 1 month.

Using Design-thinking method me and my group was challenged with a task to come up and create two connected platform ideas for "food sustainability in Denmark", so we decided to create App and info screen. This info screen is designed for large screens to be able display live tracking numbers. Data areas are meant to be customizable.

Screenshot from Figjam - hi-fi design "infoscreen for food sustainability"

For this project we had to create multifunctional cyclist enthusiast website which would cover many aspects of cycling, trips, bikes comparison, tips & tricks etc. We built this prototype using OOUX/ORCA methods, User research and thinking about design psychology. The website mainly worked as all around forum with quite limitless abilities.

Screenshot from Figjam - hi-fi design for bike enthusiast platform meant for Denmark

This is the high-fidelity prototype for the same project of "food sustainability in Denmark". As we needed to connect both platforms, we decided that Info screen’s information could also be displayed on a section within the app. User would be able to see his own food-waste statistics and overall geographical statistics across many areas.
